CSS 笔记：浮动

Posted on 2017-08-18

定位模式
常规流（Normal Flow）
浮动（Float）
绝对定位（Absolute Positioning）

浮动
浮动元素会从常规流中脱离，漂到容器左边缘或右边缘，挤开其它元素。

clear
指定元素哪一边不能与之前的浮动框相邻，取值为 left、right、both。

clearfix
在浮动元素父级之后加一个匿名 div 清除浮动。

.clearfix::after {
    content: ' ';
    display: block;
    clear: both;
    height: 0;
    overflow: hidden;
}

块级元素上下文（BFC）
BFC 可以通过 overflow: hidden 等方法创建。
BFC 可以清除浮动。
section { overflow: hidden }
BFC 可以实现双栏布局。

img { float: left }
p { overflow: hidden; padding-left: 10px }

CSS 笔记：文本样式

Posted on 2017-08-18

通用字体族
Serif 衬线体：Georgia、宋体
Sans-Serif 无衬线体：Arial、Helvetica、黑体、微软雅黑
Cursive 手写体：Caflisch Script、楷体
Fantasy 梦幻字体：Comic Sans MS
Monospace 等宽字体：Consolas、Courier、中文字体
h1 { font-family: Helvetica, sans-serif; } /* 没有 Helvetica 时，系统根据通用字体族匹配一个字体 */

font-family 使用
英文字体放在中文字体前面。（因为中文字体包含英文，会把英文设置成比如微软雅黑）
最后总是添加通用字体族。

font-size
绝对值：xx-small, x-small, small, medium, large, x-large, xx-large
相对值：larger, smaller
百分数：相对于父元素计算值
长度：px, em
初始值为 medium（由浏览器设置决定，一般 16px）

em
元素的 font-size 属性是相对于父元素的 font-size 的计算值。
其它属性是相对于元素本身的 font-size 的计算值。
h2 { font-size: 1.5em; margin-bottom: 1em; }

line-height
段落文字一般取值 1.4~1.8
p { line-height: 1.5 }

font 缩写
h1 { font: bold 14px/1.7 Helvetica, sans-serif; } /* 斜体 粗细 大小/行高 字体族 */

Web Fonts

@font-face {
    font-family: 'Lobster';
    src: local('Lobster'),
         local('Lobster-Regular'),
         url(//lib.baomitu.com/fonts/lobster/lobster-v18-latin-regular.woff2) format('woff2'),
         url(//lib.baomitu.com/fonts/lobster/lobster-v18-latin-regular.woff') format('woff');
}
h1 { font-familt: 'Lobster', cursive; }

Web fonts 链接
Google Fonts
Baomitu CDN
Font Spider

text-align
justify 是两端对齐，但最后一行/单独一行不会起作用。
p { text-align: justify; }

white-space
指定空白符如何处理，取值为 normal、nowrap、pre 等。

word-break
指定是否允许再单词中间换行，取值为 normal、break-all、break-word 等。

CSS 笔记：概念和简单选择器

Posted on 2017-08-17

CSS 语法

使用 CSS
链接，可以被浏览器缓存
<link rel="stylesheet" href="/xxx/style.css">
嵌入，省掉一个 http 请求

<style type="text/css">
    p { color: #fff; }
</style>

内联，一般不用
<p style="color:#fff">Content</p>

简单选择器
通配选择器：* { color: #fff; }
标签选择器：p { color: #fff; }
ID 选择器：#example { color: #fff; }
类选择器：.icon { color: #fff; }

其它选择器
属性选择器：input[type="password] { color: red; }
伪类选择器：a:hover { color:red }

选择器组合
直接组合 p.warning { color: red }
后代组合 article p { color: #fff }
子代组合 article>p { color: #fff }

选择器组
body, h1, h2, h3, h4, h5, h6, ul, ol, li { margin: 0; padding: 0; }

HTML 笔记：表单

Posted on 2017-08-03

GET
语义：从服务器拿数据，不会修改服务器，每次操作结果相同，会缓存
传输：数据在 url 中，一次发送

POST
语义：向服务器提交数据，会修改服务器，每次操作结果不同，不会缓存
传输：数据在 http 的请求体中，两次发送

HTTP method
GET, POST, HEAD, PUT, DELETE, OPTIONS

单行文本框
name 必须有，是向服务器发送数据对应的 key
value 是默认值
placeholder 是占位符
autofocus 在打开页面后获得焦点，比如搜索引擎输入框

input 类型
text, password, radio, checkbox, submit, file, number, range, date, month, color

非 input 类型
textarea, select

button 类型
submit, button, reset

label
input 与文字关联，提升可用性，方便盲人输入

HTML 笔记：链接与图片

Posted on 2017-08-03

https
相比 http，简单来说加密了传输过程和数据，一方面更好保护隐私，另一方面防止运营商劫持流量。

省略协议
//xxx.com/xxx，浏览器根据当前页面自动选择 http 或 https，好处是两种资源都可以加载。

省略协议和 host
浏览器根据当前页面自动补齐，实际项目中一般省略协议和 host，用绝对路径从根目录写起。

绝对路径和相对路径
绝对路径开头是 /，相对于根目录：/xxx/xxx.html。
相对路径开头不是 /，相对于当前页面所在的目录：xxx/xxx.html、./xxx/xxx.html、../../xxx.html。

页面内链接

<a href="#b">a</a>
<p id="b">b</p>

图片
alt 一定要写，特殊情况下不会丢失信息，比如盲人的读屏软件、省流量模式。
width 和 height 建议在 HTML 中加入，可以预留空白，防止加载完成后页面跳动影响用户体验。

HTML 笔记：简介

HTML 是什么？
HyperText Markup Language，使用标签来描述页面的内容和结构。

HTML 的产生
1989 年，Tim Berners-Lee，还发明了浏览器、服务器和 HTTP。

HTML 1.0
1991 年，少量标签。

HTML 2.0
1994 年，IETF，更多标签。

HTML 3.2
1997 年，W3C，HTML 3.0失败，W3C 接管 HTML 标准化。

HTML 4.01
1998 年，内容与样式分离，CSS支持。

HTML 5
2008 年，W3C HTML5 草案发布，新增语义化标签、canvas、video、audio、离线、本地储存、拖拽等。

超文本

Posted on 2017-08-02

1963 年，德特·纳尔逊(Ted Nelson)创造了术语“超文本”。1981 年，德特在他的著作中使用术语“超文本”描述了这一想法：创建一个全球化的大文档，文档的各个部分分布在不同的服务器中。通过激活称为链接的超文本项目，例如研究论文里的参考书目，就可以跳转到引用的论文。

对文本结构意义的大规模反抗始自解构主义。德里达——这位解构主义的宗师大肆宣扬文本意义的无限开放。他从结构主义的内部开始他对语言的解构，既然语言是靠内部系统的差别运作的，那么，这个差别到什么地方停止呢？他自创了一个新词：“延异”，表明这个差异的无限延宕。文本并没有固定的意义，意义从一个能指飘浮到另一个能指，闪烁在能指无穷无尽的海洋中。《人文科学话语中的结构、符号与游戏》——德里达在这篇著名论文中强调了用“游戏”来超越结构、终极、在场形而上学的阐释活动。福柯指出，为了防止文本意义的无限膨胀，“作者”常常是用来保障意义的稳定的意识形态形象。

为了彻底否弃“作者”的权威，罗兰·巴特提出了一个著名的口号：作者已死。他指出，作者只是书籍作其谓语的一个主语，作者不是个人：“言语活动认识‘主语’，不认识‘个人’”，这个主语在确定它的陈述过程之外就是空的。事实上，一个文本是由多种写作构成的，它并不存在一个单一的起源，这样的写作来自多种文化的相互对话，相互结合，相互戏仿，相互争执：“文本是由多种引证组成的编织物，它们来自文化的成千上万个源点。”废除作者与文本之间的父子关系，巴特的目的是解放文本的意义，担倡一种解除权威控制的“写作性”文本：在复合写作中，一切都在于分清，没什么需要破译的了，在每个关节点，每个层面上，结构都能被跟踪，被编织（像丝袜线团一样），然而，其底部一无所有，写作的空间应被走遍而不可穿透；写作不停地固定意义以便又不停地使之蒸发消散、使之系统地排除意义。”这样，一个统一的作者瓦解了，每个读者既是读者，又是作者。

理解 RESTful

Posted on 2017-07-19

Ivony：用 URL 定位资源，用 HTTP 描述操作。

徐磊：看 url 就知道要什么，看 http method 就知道干什么，看 http status code 就知道结果如何。

覃超：URI 使用名词而不是动词，且推荐用复数。

BAD

  • /getProducts
  • /listOrders
  • /retrieveClientByOrder?orderld=1

GOOD

  • GET /products: will return the list of all products
  • POST /products: will add a product to the collection
  • GET /products/4: will retrieve product #4
  • PATCH/PUT /products/4 will update product #4
