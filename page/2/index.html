JS 笔记：函数

Posted on 2017-08-25

函数声明和函数表达式
函数声明会提升，函数表达式不会。

console.log([typeof add, typeof sub]); // ["function", "undefined"]
function add(x, y) { return x + y; }
var sub = function(x, y) { return x - y; }

闭包
闭包通常用来创建内部变量，使得这些变量不能被外部随意修改，同时又可以通过指定的函数接口来操作。

闭包作用域
用IIFE产生的闭包模拟块级作用域：

for(var i = 0; i < 10; i++) {
    (function(i) {
        setTimeout(function() { console.log(i); }, 100 * i);
    })(i)
}

es6:

for(let i = 0; i < 10; i++) {
    setTimeout(function() { console.log(i); }, 100 * i);
}

this
全局范围：指全局对象，浏览器中指 window。
对象的方法：指这个对象。
构造函数：指新对象。
call/apply：由参数指定。

异步
Timer
事件
获取数据 API
Promise
其它异步模型

数据抽象和过程抽象
数据抽象：
define: f -> dataFormF = f(data)
define: g -> result = g(dataFormF)
过程抽象：
define: f -> dateFormF = f(data)
define: g -> k = g(f), result = k(data)

高阶函数
把函数作为参数传入，这样的函数称为高阶函数，函数式编程就是指这种高度抽象的编程范式。
（也可以理解为操作函数的函数）

函数的本质
封装过程，开放接口。
好的程序涉及是面向接口编程，而不是面向实现编程。
（需要使用者理解实现的代码不是好代码）

JS 笔记：变量、值与类型

Posted on 2017-08-25

数据类型
number, strign, boolean, null, undefined, object

变量提升
变量提升即将变量声明提升到它所在作用域的最开始的部分。

console.log(global); // undefined
var global = 'global';
console.log(global); // global
function fn () {
　　console.log(a); // undefined
　　var a = 'aaa';
　　console.log(a); // aaa
}
fn();

之所以会是以上的打印结果，是由于变量提升，实际上上面的代码是按照以下来执行的：

var global; // 变量提升，全局作用域范围内，此时只是声明，并没有赋值
console.log(global); // undefined
global = 'global'; // 此时才赋值
console.log(global); // 打印出global
function fn () {
　　var a; // 变量提升，函数作用域范围内
　　console.log(a); // undefined
　　a = 'aaa';
　　console.log(a); // aaa
}
fn();

Boolean
建议采用严格比较，可以通过 !! 将非 boolean 值转为 boolean。
布尔操作符 && 和 || 并不会转换类型。
比较操作符总是返回 boolean 类型。

var result = 1 && 2;
console.log([result, !!result]); // [2, true]

Number
数值范围：Infinity、Number.MAX_VALUE、Number.MIN_VALUE
浮点数精度：console.log(0.2 + 0.4); // 0.6000000000000001

String
隐式类型转换：console.log([1+2, '1'+2, '1'-0+2]); // [3, "12", 3]
转整数：console.log(parseInt('100abc', 2)); // 100
转浮点数：console.log(parseFloat('12.3e5xx'); // 1230000;

对象的拷贝
直接引用：let obj2 = obj1; // 改变 obj2，原对象不会变
浅拷贝：let obj2 = Object.assign({}, obj1); // 改变 obj2，原对象属性不会变，子对象会变
深拷贝：递归函数实现。 // 原对象不会变

原型链
__proto__ 构建原型链：

var a = {x: 1}, b = {y: 2}, c = {z: 3};
b.__proto__ = a;
c.__proto__ = b;

使用 Object.create 构建原型链：

var a = {x: 1};
var b = Object.create(a);
b.y = 2;
var c = Object.create(b);
c.z = 3;

使用构造器方式：（标准）

function A() { this.x = 1; }
function B() { this.y = 2; }
B.prototype = new A();
function C() { this.z = 3; }
C.prototype = new B();
var c = new C();

内置类型
Object, Function, Array, Date, Math, Regex, Error, Promise…

CSS reset
设置 HTML 标签的默认样式，使其在各个浏览器表现基本一致，让默认样式归零。
常见设置有 marging、padding、inherit、outline 等。

normalize.css
设置 HTML 标签的默认样式，使其在各个浏览器表现基本一致，保留标签的默认样式。

CSS 模块原则
面向对象（OOCSS）
单一职责原则（SRP）
开闭原则
Don’t Repeat Yourself（DRY）

面向对象（OOCSS）
把基础的样式抽象出来，不一样的地方做继承。

.btn {
    display: inline-block;
    padding: 0.36em 0.8em;
    margin-right: 0.5em;
    border: none;
    border-radius: 0.3em;
    cursor: pointer;
}
.btn--primary {
    background-color: #09c;
    color: #fff;
}

OOCSS 原则
结构和皮肤分离：.bn .btn–primary .btn–info .btn–danger
容器和内容分离：.header .btn { color: #fff; } /* 不推荐这样写 */

media object

.media
    .media__image
        .media_object
    .media__body
        .media__title
        .media__description

单一职责原则
尽可能细地拆分成可独立复用的组件，通过组合方式使用多个组件。

开闭原则
对扩展开放，对修改关闭。

.box { display: block; padding: 10px; }
.content .box { padding: 20px; } /* bad example */
.box--large { padding: 20px; } /* better example */

DRY 原则

/* bad example */
.selector {
    border-top: 1px solid #fcc;
    border-left: 1px solid #fcc;
    border-bottom: 1px solid #fcc;
}
/* better example */
.selector {
    border: 1px solid #fcc;
    border-right: none;
}

命名
基于功能：.button/.form/.list/.external-link/.tab-item/（优先使用）
基于内容：.news/.user-info/.help/.contact-me（适合中小型网站）
基于视觉：.round-image/.nowrap（适合大型网站）
建议使用抽象的命名：

CSS 笔记：兼容性

Posted on 2017-08-18

兼容性问题
浏览器不支持该特性。
某些特定条件下出发浏览器 bug。

解决方案

  • 低版本浏览器没有这个特性可以接受吗？
    border-radius 不支持时，没有圆角。
    box-shadow 不支持时，没有阴影。
  • 可以使用效果稍微差一些的替代方案吗？
    min-height: 100vh，用 min-height: 800px。
  • 可以使用一些特带方案吗？
    opacity: 0.5 在 IE 下用 filter: alpha(opacity=50)。
  • 可以使用 JavaScript 让浏览器支持吗？
    使用 html5shim.js 让 IE6~8 支持新标签。
    使用 DD_belatedPNG.js 让IE6 支持半透明 png 图片。
  • 更换实现方式。

Chrome/Safari/Opera: -webkit-
Microsoft: -ms-
Mozilla: -moz-

不同浏览器使用不同的样式
@supports、层叠、条件注释、浏览器怪癖

CSS3 属性
IE6-7 不支持
IE8 部分支持
IE9 基本都支持

IE6、7 不支持
IE6、7 很多都不支持

IE8 不支持
background-size：推荐在 IE8 及以下使用固定宽度布局。
border-radius
box-shadow
opacity：filter: alpha(opacity=50)
rgba、hsl、hsla：一般场景下，使用相近的不透明颜色代替。
rem/vh/vw/calc：降级为固定宽度。

IE9 不支持
transition 与 animation：可以接受降级，如果不能接受就用 JavaScript。

控制 IE 模式（<=10）
meta 标签控制进入哪种文档模式。

<!-- 使用IE7模式渲染-->
<meta http-equiv="X-UA-Compatible" content="IE=7">
<!-- 使用最新引擎-->
<meta http-equiv="x-ua-compatible" content="IE=edge">

兼容写法工具

让 IE 系列支持 html5 语义化标签。

测试兼容性

最后的建议
推荐使用新的写法，通过 Polyfill 和 Autoprefixer 等工具帮助处理。
不要因为兼容性而牺牲新特性，使用旧的写法。

CSS 笔记：布局

Posted on 2017-08-18

布局方法
float, position, inline-block, table, flex, grid

水平居中
行级元素：text-align: center
块级元素：margin: auto

max-width
大部分图文内容的网页，布局都可以用 max-width 加 margin: auto。

垂直居中
单行文字：line-height
行级盒子：vertical-align: middle
绝对定位：top: 50%; left: 50%

position
用 absolute 和 margin 进行两栏布局。

main { position: relative; }
article { margin-left: 10em; }
aside { position: absolute; width: 10em; top: 0; left: 0; }

table
用 display 模拟表格布局，宽度自适应。不要使用表格标签布局。

main { display: table; }
article { display: table-cell; }
aside { display: table-cell; }

可用于实现居中效果。
.container { display: table-cell; text-align: center; vertical-align: middle; }

display: flex
将元素变为 Flexbox。

flex-direction
子元素排列方向，取值为 row、row-reverse、column、column-reverse。

flex-grow
元素扩展（占剩余空间）的能力，默认值为 0。

flex-shrink
元素收缩的能力，默认值为 1。

flex-wrap
元素在主轴方向排放时，能否换行，取值为 nowrap、wrap、wrap-reverse。

justify-content
子元素沿主轴方向的摆放，取值为 flex-start、flex-end、center、space-between、space-around。

align-items
子元素在侧轴方向的对齐方式，取值为 flex-start、flex-end、center、baseline、stretch。

align-self
对单独的元素设置对齐方式。

align-content
多行内容在容器内侧轴方向的对齐，取值为 flex-start、flex-end、center、space-between、space-around、stretch。

order
指定摆放时的顺序，默认值为 0。
