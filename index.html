在 create-react-app 中使用 antd

Posted on 2017-09-15

安装 create-react-app
$ npm install -g create-react-app 安装 create-react-app
$ create-react-app new-app 创建新项目
$ cd antd-demo 进入项目

更改 create-react-app 默认配置
$ npm install react-app-rewired --dev 安装 react-app-rewired 插件
修改 package.json 里的启动配置：

"scripts": {
-   "start": "react-scripts start",
+   "start": "react-app-rewired start",
-   "build": "react-scripts build",
+   "build": "react-app-rewired build",
-   "test": "react-scripts test --env=jsdom",
+   "test": "react-app-rewired test --env=jsdom",
}

在项目根目录创建一个 config-overrides.js 用于修改默认配置：

module.exports = function override(config, env) {
  // do stuff with the webpack config...
  return config;
};

安装 antd
$ npm install antd babel-pluging-import --save 安装 antd 和 babel-pluging-import 插件
修改 config-overrides.js 配置：

+ const { injectBabelPlugin } = require('react-app-rewired');

  module.exports = function override(config, env) {
+   config = injectBabelPlugin(['import', { libraryName: 'antd', style: 'css' }], config);
    return config;
  };

开始使用
修改 src/App.js，引入 antd 的按钮插件：

    import React, { Component } from 'react';
    import './App.css';
+   import { Button } from 'antd';

    class App extends Component {
      render() {
        return (
          <div className="App">
            <Button type="primary">Button</Button>
          </div>
        );
      }
    }

    export default App;

自定义主题（不必要）
$ npm install react-app-rewire-less --dev 安装 react-app-rewire-less 插件

修改 config-overrides.js 配置：

  const { injectBabelPlugin } = require('react-app-rewired');
+ const rewireLess = require('react-app-rewire-less');

  module.exports = function override(config, env) {
-   config = injectBabelPlugin(['import', { libraryName: 'antd', style: 'css' }], config);
+   config = injectBabelPlugin(['import', { libraryName: 'antd', style: true }], config);
+   config = rewireLess(config, env, {
+     modifyVars: { "@primary-color": "#1DA57A" },
+   });
    return config;
  };

修改后重启 npm start，如果看到一个绿色的按钮就说明配置成功了。

JS 笔记：常用设计模式和组件开发

Posted on 2017-08-25

模式1：Behavior
从交互需求出发设计一组“动作行为”，在主题对象上应用和扩展这些行为。

模式2：Adapter
追对不同底层实现统一接口的外部接口，让业务层面的代码可以适应不同的底层实现。

模式3：Obsever
同步多个对象的状态变化，解耦各个对象之间的依赖。

模式4：Mediator
将命令传递给所有订阅者，与订阅者们维持依赖关系，从而让订阅者彼此之间相互独立。

模式5：Omposite
对已有组件/库/功能的组合使用，提升代码复用性的最常用手段。

模式6：Partial Application
Function.prototype.bind，部分应用/固化参数

模式7：Decorate
在原有 API 基础上增加额外的功能，不改变底层库的情况下扩展功能的一种模式。

模式8：Proxy
保持原有 API 功能不变，改变执行过程和结果，不改变底层库的情况下获得不同执行结果的手段。

模式9：Aspect
对代码执行的过程进行监测和修改，不改变代码主体执行流程的情况下控制执行过程。
（preventDefault()、Symblos…）

Decorate、Proxy 和 Aspect 的联系
Decorate 强调的是声明和定义。
Proxy 强调的是功能。
Aspect 强调的是运行过程。
动态语言可以用 Decorate 来实现 Proxy 和 Aspect。

模式10：Lazy load
一些加载耗时的资源进行分段、异步加载，提升前端性能，减少相应用户的时间，某些情况下减少网页流量和带宽消耗。

总结
我们所做的一切都是为了提升抽象程度和灵活性，从而改善我们的代码质量，降低维护成本。

JS 笔记：DOM & BOM

Posted on 2017-08-25

事件绑定
行内绑定：<input type="button" onclick="function();" />
DOM 绑定：btn.onclick = function();
事件监听：btn.addEventListener('click', function());

事件流
动画演示

捕获阶段
当我们在 DOM 树的某个节点发生了一些操作（例如单击、鼠标移动上去），就会有一个事件发射过去。这个事件从 Window 发出，不断经过下级节点直到目标节点。在到达目标节点之前的过程，就是捕获阶段（Capture Phase）。
btn.addEventListener('click', function(), true); // 监听捕获阶段，从顶层到目标依次处理事件

目标阶段
事件到了事件触发目标节点那里，最终在目标节点上触发这个事件，就是目标阶段。

冒泡阶段
当事件达到目标节点之后，就会沿着原路返回，由于这个过程类似水泡从底部浮到顶部，所以称作冒泡阶段。
btn.addEventListener('click', function()); // 默认监听冒泡阶段，从目标到顶层依次处理事件

停止事件冒泡
event.stopPropagation(); // 监听事件节点，只在目标节点处理事件

事件代理
因为事件有冒泡机制，所有子节点的事件都会顺着父级节点跑回去，所以我们可以通过监听父级节点来实现监听子节点的功能，这就是事件代理。
优势：
1.减少事件绑定，提升性能。
2.动态变化的 DOM 结构，仍然可以监听。

UI 组件设计
1.结构设计
2.API 设计
3.控制流设计

浏览器常用 API
alert/confirm, Location, navigator.userAgent, Cookie, Storage, XHR(Ajax), WebSockets, Canvas

Cookie
网站跟踪用户行为，服务器用以标记 Session
少量数据，跟随 HTTP 请求（header）发送
可设置过期时间
网站 Cookie 数量有限制
支持 http-only
JavaScript 原生的 Cookie API 不好用

Storage
可以储存的数据量比较大，但仍有限制，别滥用
数据格式为字符串，可以用 JOSN 解析
有 LocalStorage 和 SessionStorage 两种类型，默认无过期时间
并不会随着 HTTP 请求发给服务器

Canvas
绘制图形
处理图片（必须是同域）
高性能动画
2D 游戏

JS 笔记：函数

Posted on 2017-08-25

函数声明和函数表达式
函数声明会提升，函数表达式不会。

console.log([typeof add, typeof sub]); // ["function", "undefined"]
function add(x, y) { return x + y; }
var sub = function(x, y) { return x - y; }

闭包
闭包通常用来创建内部变量，使得这些变量不能被外部随意修改，同时又可以通过指定的函数接口来操作。

闭包作用域
用IIFE产生的闭包模拟块级作用域：

for(var i = 0; i < 10; i++) {
    (function(i) {
        setTimeout(function() { console.log(i); }, 100 * i);
    })(i)
}

es6:

for(let i = 0; i < 10; i++) {
    setTimeout(function() { console.log(i); }, 100 * i);
}

this
全局范围：指全局对象，浏览器中指 window。
对象的方法：指这个对象。
构造函数：指新对象。
call/apply：由参数指定。

异步
Timer
事件
获取数据 API
Promise
其它异步模型

数据抽象和过程抽象
数据抽象：
define: f -> dataFormF = f(data)
define: g -> result = g(dataFormF)
过程抽象：
define: f -> dateFormF = f(data)
define: g -> k = g(f), result = k(data)

高阶函数
把函数作为参数传入，这样的函数称为高阶函数，函数式编程就是指这种高度抽象的编程范式。
（也可以理解为操作函数的函数）

函数的本质
封装过程，开放接口。
好的程序涉及是面向接口编程，而不是面向实现编程。
（需要使用者理解实现的代码不是好代码）

JS 笔记：变量、值与类型

Posted on 2017-08-25

数据类型
number, strign, boolean, null, undefined, object

变量提升
变量提升即将变量声明提升到它所在作用域的最开始的部分。

console.log(global); // undefined
var global = 'global';
console.log(global); // global
function fn () {
　　console.log(a); // undefined
　　var a = 'aaa';
　　console.log(a); // aaa
}
fn();

之所以会是以上的打印结果，是由于变量提升，实际上上面的代码是按照以下来执行的：

var global; // 变量提升，全局作用域范围内，此时只是声明，并没有赋值
console.log(global); // undefined
global = 'global'; // 此时才赋值
console.log(global); // 打印出global
function fn () {
　　var a; // 变量提升，函数作用域范围内
　　console.log(a); // undefined
　　a = 'aaa';
　　console.log(a); // aaa
}
fn();

Boolean
建议采用严格比较，可以通过 !! 将非 boolean 值转为 boolean。
布尔操作符 && 和 || 并不会转换类型。
比较操作符总是返回 boolean 类型。

var result = 1 && 2;
console.log([result, !!result]); // [2, true]

Number
数值范围：Infinity、Number.MAX_VALUE、Number.MIN_VALUE
浮点数精度：console.log(0.2 + 0.4); // 0.6000000000000001

String
隐式类型转换：console.log([1+2, '1'+2, '1'-0+2]); // [3, "12", 3]
转整数：console.log(parseInt('100abc', 2)); // 100
转浮点数：console.log(parseFloat('12.3e5xx'); // 1230000;

对象的拷贝
直接引用：let obj2 = obj1; // 改变 obj2，原对象不会变
浅拷贝：let obj2 = Object.assign({}, obj1); // 改变 obj2，原对象属性不会变，子对象会变
深拷贝：递归函数实现。 // 原对象不会变

原型链
__proto__ 构建原型链：

var a = {x: 1}, b = {y: 2}, c = {z: 3};
b.__proto__ = a;
c.__proto__ = b;

使用 Object.create 构建原型链：

var a = {x: 1};
var b = Object.create(a);
b.y = 2;
var c = Object.create(b);
c.z = 3;

使用构造器方式：（标准）

function A() { this.x = 1; }
function B() { this.y = 2; }
B.prototype = new A();
function C() { this.z = 3; }
C.prototype = new B();
var c = new C();

内置类型
Object, Function, Array, Date, Math, Regex, Error, Promise…
