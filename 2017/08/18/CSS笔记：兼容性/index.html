CSS 笔记：兼容性

Posted on 2017-08-18

兼容性问题
浏览器不支持该特性。
某些特定条件下出发浏览器 bug。

解决方案

  • 低版本浏览器没有这个特性可以接受吗？
    border-radius 不支持时，没有圆角。
    box-shadow 不支持时，没有阴影。
  • 可以使用效果稍微差一些的替代方案吗？
    min-height: 100vh，用 min-height: 800px。
  • 可以使用一些特带方案吗？
    opacity: 0.5 在 IE 下用 filter: alpha(opacity=50)。
  • 可以使用 JavaScript 让浏览器支持吗？
    使用 html5shim.js 让 IE6~8 支持新标签。
    使用 DD_belatedPNG.js 让IE6 支持半透明 png 图片。
  • 更换实现方式。

浏览器前缀
浏览器产商为了实验新特性，在属性名前加前缀。
Chrome/Safari/Opera: -webkit-
Microsoft: -ms-
Mozilla: -moz-

不同浏览器使用不同的样式
@supports、层叠、条件注释、浏览器怪癖

CSS3 属性
IE6-7 不支持
IE8 部分支持
IE9 基本都支持

IE6、7 不支持
IE6、7 很多都不支持

IE8 不支持
background-size：推荐在 IE8 及以下使用固定宽度布局。
border-radius
box-shadow
opacity：filter: alpha(opacity=50)
rgba、hsl、hsla：一般场景下，使用相近的不透明颜色代替。
rem/vh/vw/calc：降级为固定宽度。

IE9 不支持
transition 与 animation：可以接受降级，如果不能接受就用 JavaScript。

控制 IE 模式（<=10）
meta 标签控制进入哪种文档模式。

<!-- 使用IE7模式渲染-->
<meta http-equiv="X-UA-Compatible" content="IE=7">
<!-- 使用最新引擎-->
<meta http-equiv="x-ua-compatible" content="IE=edge">

兼容写法工具
Autoprefixer

html5shiv.js
让 IE 系列支持 html5 语义化标签。

测试兼容性
虚拟机
BrowserStack

Polyfill
使用代码帮助浏览器实现它尚未支持的特性。
box-sizing-polyfill
flexibility
selectivizr
CSS3 PIE
cssSandpaper

最后的建议
推荐使用新的写法，通过 Polyfill 和 Autoprefixer 等工具帮助处理。
不要因为兼容性而牺牲新特性，使用旧的写法。
